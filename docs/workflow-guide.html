Workflow Guide

Copy these prompts in order — from product definition to deployment. Each phase builds on the previous one. Conditional phases (Design, Debugging) can be skipped when not needed.

Handoff Rule: Each phase must pass the next phase: document path, decisions, assumptions, open questions, acceptance criteria, and risks.

Architecture

Tips: Jangan lewati — bahkan untuk fitur kecil, PRD singkat membantu agent memahami konteks. Output berupa PRD.md + decision ledger.

Code Review

Tips: Satu task instruction per task. Jangan overload agent dengan banyak task sekaligus. Review evidence sebelum move ke task berikutnya.

Kondisional. Hanya ketika ada error, bug, test gagal, atau perilaku tidak sesuai harapan. Workflow ini membantu root cause analysis + verified fix.
